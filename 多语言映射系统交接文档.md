# OpenPnP 多语言映射系统交接文档

## 一、系统概述

本系统为 OpenPnP 实现了一套基于唯一 ID 的集中式多语言映射框架，支持问题/解决方案文本、类名等内容的中英文映射，并提供可视化的映射表管理界面。

---

## 二、核心设计思路

### 2.1 架构设计

采用双层映射架构：

| 映射类型 | 用途 | 数据源 |
|---------|------|--------|
| **类名映射 (ClassMapping)** | 设备组件类名的翻译 | `ClassTitleRegistry.java` 静态注册 |
| **文本映射 (TextMapping)** | 问题/解决方案文本的翻译 | `ClassTitleRegistry.java` 静态注册 |
| **模式映射 (TextMappingPattern)** | 含动态变量文本的翻译 | 正则表达式匹配 |

### 2.2 PID 标识机制

每个映射项分配唯一 ID（如 `ID_001`），映射启用时在文本前添加 `(PID:xxx)` 前缀，便于追踪和搜索：

```
启用映射前: "主轴"
启用映射后: "(PID:015) 控制器轴"
```

### 2.3 状态持久化

使用 Java Preferences API 保存以下状态：
- 映射启用/禁用状态
- 表格列宽配置

## 四、功能实现

### 4.1 映射启用开关

- **位置**：映射表对话框顶部复选框
- **功能**：控制全局映射是否生效
- **持久化**：重启后保持用户设置

### 4.2 类名映射

- **覆盖范围**：所有实现 `Solutions.Subject` 接口的设备组件
- **自动附加名称**：支持在类名后自动附加设备名称（如 `(PID:003) 吸嘴 N1`）
- **语言切换**：根据软件语言设置自动切换中英文

### 4.3 文本映射

- **静态文本**：精确匹配的问题和解决方案文本（约 180+ 条）
- **动态文本**：含设备名的文本使用正则表达式模式匹配（如 `Set the manual nozzle tip change location for N1`）

### 4.4 映射表管理界面

| 功能 | 说明 |
|------|------|
| **搜索** | 支持按 ID、原文、翻译、分类搜索 |
| **语言切换** | 界面元素随软件语言自动切换 |
| **列宽持久化** | 调整列宽后下次打开自动恢复 |
| **单元格编辑** | ID 和原文不可编辑，翻译和分类可编辑 |

### 4.5 搜索功能

- 在映射表中输入 PID 或文本内容即可搜索
- 支持中英文混合搜索

## 五、扩展指南

### 5.1 添加新的类名映射

在 `ClassTitleRegistry.java` 的静态块中添加：

```java
addMapping(new ClassMapping("ID_XXX", "完整类名", "简单类名", 
        "翻译键", "英文标题", "中文标题", 是否附加名称, "分类"));
```

### 5.2 添加新的文本映射

```java
// 静态文本映射
addTextMapping(new TextMapping("ID_XXX", "原文", "英文翻译", "中文翻译"));

// 动态文本映射（含变量）
addTextMappingPattern(new TextMappingPattern("ID_XXX", 
        "^正则表达式$", "英文模板{0}", "中文模板{0}"));
```

### 5.3 在代码中使用映射

```java
// 类名映射
String title = ClassTitleRegistry.getTitle(object);

// 文本映射
String text = ClassTitleRegistry.getTextMapping("原始文本");
```
