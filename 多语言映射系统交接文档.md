# OpenPnP 多语言映射系统交接文档

## 一、系统概述

本系统为 OpenPnP 实现了一套**基于配置文件的集中式多语言映射框架**，支持设备类名、问题/解决方案文本的中英文映射，并提供可视化的映射表管理界面。

### 核心特点

- **映射数据与代码完全分离** — 修改映射无需重新编译
- **默认值与用户自定义分离** — 升级不覆盖用户配置
- **运行时动态加载** — 默认配置 + 用户覆盖双层机制

## 二、架构设计

### 2.1 数据流

```
┌─────────────────────────────────────────────────────────┐
│                   启动加载流程                            │
│                                                         │
│  1. 从 classpath 加载默认配置文件                          │
│     src/main/resources/org/openpnp/                     │
│       ├── class-mappings.properties    (类映射)           │
│       ├── text-mappings.properties     (文本映射)          │
│       └── text-mapping-patterns.properties (模式映射)      │
│                                                         │
│  2. 从用户目录加载覆盖配置                                 │
│     ~/.openpnp/mappings/                                │
│       ├── class-mappings.properties    (用户覆盖)          │
│       ├── text-mappings.properties     (用户覆盖)          │
│       └── text-mapping-patterns.properties (用户覆盖)      │
│                                                         │
│  3. 用户覆盖值替换默认值 → 内存中构建映射索引                │
└─────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────┐
│                   运行时使用                               │
│                                                         │
│  ClassTitleRegistry.getTitle(obj)       → 类名映射       │
│  ClassTitleRegistry.getTextMapping(text) → 文本映射      │
│  ClassTitleRegistry.getTitleFromDisplayText() → 显示文本  │
└─────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────┐
│                   编辑保存流程                             │
│                                                         │
│  映射表 UI → 编辑单元格 → 点击保存/确定                    │
│                     → ClassTitleRegistry.saveAll()       │
│                     → 写入 ~/.openpnp/mappings/          │
│                     → 下次启动自动加载                     │
└─────────────────────────────────────────────────────────┘
```

### 2.2 映射类型

| 映射类型 | 用途 | PID 范围 | 存储文件 |
|---------|------|---------|---------|
| **类名映射 (ClassMapping)** | 设备组件类名的翻译 | 001-199 | `class-mappings.properties` |
| **文本映射 (TextMapping)** | 问题/解决方案文本的翻译 | 200-349 | `text-mappings.properties` |
| **模式映射 (TextMappingPattern)** | 含动态变量文本的翻译 | 350-399 | `text-mapping-patterns.properties` |

### 2.3 PID 标识机制

每个映射项分配唯一 PID（纯数字），映射启用时在文本前添加 `(PID:xxx)` 前缀：

```
启用映射前: "Controller Axis x"
启用映射后: "(PID:015) 控制器轴 x"
```

## 三、文件结构

### 3.1 核心文件

| 文件路径 | 说明 |
|---------|------|
| `src/main/java/org/openpnp/ClassTitleRegistry.java` | **映射注册表核心类** - 加载/查询/保存映射 |
| `src/main/java/org/openpnp/gui/ClassTitleMappingDialog.java` | **映射表管理对话框** - 可视化编辑界面 |
| `src/main/java/org/openpnp/Translations.java` | **国际化工具类** - 语言切换支持 |
| `src/main/resources/org/openpnp/class-mappings.properties` | **类名映射数据**（默认配置） |
| `src/main/resources/org/openpnp/text-mappings.properties` | **文本映射数据**（默认配置） |
| `src/main/resources/org/openpnp/text-mapping-patterns.properties` | **模式映射数据**（默认配置） |

### 3.2 被修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `src/main/java/org/openpnp/model/Solutions.java` | `getSubjectText()` 调用 `ClassTitleRegistry.getTitle()`；表格渲染使用 `getTextMapping()` |
| `src/main/java/org/openpnp/gui/components/IssuePanel.java` | 使用映射后的文本显示问题和解决方案 |
| `src/main/java/org/openpnp/gui/MachineSetupPanel.java` | 设备树节点使用 `getTitleFromDisplayText()` |
| `src/main/resources/org/openpnp/translations.properties` | 添加映射表界面翻译键 |
| `src/main/resources/org/openpnp/translations_zh_CN.properties` | 添加映射表界面中文翻译 |

### 3.3 配置文件格式

**class-mappings.properties**
```properties
# PID|fullClassName|simpleName|englishTitle|chineseTitle|appendName|category
001|org.openpnp.machine.reference.ReferenceMachine|ReferenceMachine|Machine|机器|false|Machine
002|org.openpnp.machine.reference.ReferenceHead|ReferenceHead|Head|贴装头|true|Machine
```

- `appendName`: `true` 表示在标题后附加设备名称（如"吸嘴 N1"）
- `category`: 分类标识，用于UI分组

**text-mappings.properties**
```properties
# PID|originalEnglishText|englishTranslation|chineseTranslation
200|Axis is not assigned to a driver.|Axis is not assigned to a driver.|轴未分配给驱动器。
201|Assign a driver.|Assign a driver.|分配一个驱动器。
```

**text-mapping-patterns.properties**
```properties
# PID|regex|englishTemplate|chineseTemplate
350|^Set the manual nozzle tip change location for (.*?)\.$|Set the manual nozzle tip change location for {0}.|设置{0}的手动吸嘴尖更换位置。
```

- 模式匹配使用 Java 正则表达式
- `{0}`, `{1}` 等占位符会被正则捕获组替换

## 四、核心类说明

### 4.1 ClassTitleRegistry

位于 `org.openpnp` 包，提供静态方法接口：

| 方法 | 说明 |
|------|------|
| `getTitle(Object obj)` | 获取对象的映射标题（带PID前缀） |
| `getTextMapping(String text)` | 获取文本的映射版本（精确匹配 → 模式匹配） |
| `getTitleFromDisplayText(Object obj, String displayText)` | 根据显示文本获取映射标题 |
| `getAllMappings()` | 获取所有类映射 |
| `getAllTextMappings()` | 获取所有文本映射 |
| `isMappingEnabled()` / `setMappingEnabled()` | 控制映射启用开关 |
| `saveAll()` | 将所有映射保存到用户目录 |
| `getMapping(Class<?> clazz)` | 根据类获取映射 |
| `getMappingById(String id)` / `getTextMappingById(String id)` | 根据ID获取映射 |

### 4.2 内部数据结构

```java
// 类名映射
ClassMapping {
    String id;             // "ID_001"
    String className;      // 完整类名
    String simpleName;     // 简单类名
    String translationKey; // 翻译键
    String englishTitle;   // 英文标题
    String chineseTitle;   // 中文标题
    boolean appendName;    // 是否附加设备名
    String category;       // 分类
}

// 文本映射
TextMapping {
    String id;              // "ID_200"
    String text;            // 原文
    String englishTranslation; // 英文翻译
    String chineseTranslation; // 中文翻译
}

// 模式映射
TextMappingPattern {
    String id;              // "ID_350"
    Pattern pattern;        // 正则表达式
    String englishTemplate; // 英文模板
    String chineseTemplate; // 中文模板
}
```

### 4.3 ClassTitleMappingDialog

映射表管理对话框，位于 `org.openpnp.gui` 包：

**界面结构**：
- 顶部工具栏：启用映射复选框、语言切换、搜索框
- 选项卡：类映射 / 文本映射
- 底部按钮：保存、确定、取消

**功能特性**：
- 表格列宽可调并自动保存
- ID和原文列不可编辑，翻译列可编辑
- 支持按 PID 或文本内容搜索
- 界面语言随软件语言自动切换

## 五、映射查询逻辑

### 5.1 类名映射 `getTitle(obj)`

```
1. obj == null → 返回 ""
2. 映射未启用 → 返回默认标题（SimpleName + Name/ID）
3. 查找类映射 → 未找到 → 返回默认标题
4. 根据当前语言选择英文/中文标题
5. appendName=true 且 obj 实现了 Named → 附加设备名
6. 返回 "(PID:xxx) 标题 [名称]"
```

### 5.2 文本映射 `getTextMapping(text)`

```
1. text == null/空 → 返回原文本
2. 映射未启用 → 返回原文本
3. 精确匹配 → 返回 "(PID:xxx) 翻译文本"
4. 模式匹配 → 用正则捕获组替换模板占位符 → 返回 "(PID:xxx) 处理结果"
5. 都未匹配 → 返回原文本
```

## 六、用户使用指南

### 6.1 打开映射表

- 菜单路径：**Tools → Mapping Table**
- 或通过主界面工具栏相关按钮

### 6.2 编辑映射

1. 在表格中直接编辑翻译、分类等可编辑列
2. 点击 **Save** 保存到用户目录
3. 重启程序后自动加载修改后的映射

### 6.3 启用/禁用映射

- 勾选映射表顶部的 **"Enable Mapping"** 复选框
- 状态自动保存，重启后保持

### 6.4 搜索映射

- 在搜索框中输入 PID（如 `015`）或文本内容
- 表格自动过滤显示匹配项

## 七、扩展指南

### 7.1 添加新的类名映射

编辑 `class-mappings.properties`，按格式添加一行：

```properties
030|org.openpnp.machine.reference.MyNewFeeder|MyNewFeeder|My Feeder|我的送料器|true|Feeder
```

### 7.2 添加新的文本映射

编辑 `text-mappings.properties`：

```properties
280|Some new issue text.|Some new issue text.|某个新问题文本。
```

### 7.3 添加新的模式映射

编辑 `text-mapping-patterns.properties`：

```properties
380|^Configure the (.*?) for (.*?)\.$|Configure the {0} for {1}.|为{1}配置{0}。
```

### 7.4 注意事项

- PID 范围：类映射 001-199，文本映射 200-349，模式映射 350-399
- PID 必须唯一，不要重复
- 配置文件使用 UTF-8 编码
- 修改默认配置文件（`src/main/resources/` 目录下）需要重新编译打包才会生效
- 用户自定义修改保存到 `~/.openpnp/mappings/`，立即生效无需编译

## 八、运行方式

### 8.1 编译

```bash
cd openpnp
mvn compile -DskipTests
```

### 8.2 运行

```bash
java --add-opens=java.base/java.lang=ALL-UNNAMED \
     --add-opens=java.desktop/java.awt=ALL-UNNAMED \
     --add-opens=java.desktop/java.awt.color=ALL-UNNAMED \
     -cp "target/classes;target/lib/*" \
     org.openpnp.Main
```

### 8.3 快速启动脚本

项目根目录提供 `openpnp.bat`（Windows）和 `openpnp.sh`（Linux/Mac）。
